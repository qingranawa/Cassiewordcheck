# ADR-003: 使用 JSON 而非 SQLite 做持久化

## 背景

应用需要持久化两类数据：用户设置（9 个小字段，启动时加载）和检查历史（最多 50 条，每 3 分钟自动保存）。数据量极小（< 100KB）。

## 决策

使用 `System.Text.Json` 以 JSON 文件做持久化（`data/appsettings.json` 和 `data/history.json`）。无外部数据库依赖。

## 被拒绝的替代方案

- **SQLite（via Microsoft.Data.Sqlite）**：引入原生依赖和构建复杂度。对 <100KB 的扁平对象数据（无关系查询）来说大材小用。
- **二进制序列化**：不可读。版本管理困难——模式变更会静默失败。JSON 允许手动编辑和调试。
- **Windows 注册表**：不透明，难以备份或迁移。不适合历史记录。

## 后果

- 每次保存需读写整个文件——小文件可接受（<1ms 每次操作）
- 无并发访问保护——应用是单实例，不会产生冲突
- JSON 反序列化失败时静默恢复（回退到默认值）
- 文件可读，方便调试和手动编辑
